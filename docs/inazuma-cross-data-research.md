- ポジション（FW / MF / DF / GK）

- 不動明王・豪炎寺修也・吹雪士郎の具体的なパッシブスキル

### ⚠️ 非公式情報（攻略サイト記載、要検証）

- 星2の初期覚醒ランクについて：
  - ユーザー実測では「normal player+」スタート
  - Gamerchでは「normal player」スタートとも読める記載あり（表フォーマットの解釈による）
  - → **ユーザー実測データを優先して記載、Gamerchは誤記録の可能性あり**
- 星3の top player+ → legendary player の必要属性のかけら数（Gamerchでは「確認中」）
  - ユーザー実測で 600個と確認済み → 確定
- 星2の legendary player → legendary player+ の必要覚醒魂数（Gamerchでは「確認中」）
  - ユーザー実測で 15個と確認済み → 確定
- 人脈ボード入手キャラの覚醒素材の違い → **要検証**
- ガチャ限以外（イベント配布等）の覚醒ルート → **要検証**
- 各キャラのチーム所属（一部不明）

### ❌ 未確認情報

- 星1キャラのGK一覧（攻略サイトに掲載なし）
- パッシブスキルの完全一覧（全キャラ分は未収集）

## 11. パッシブスキルのデータ構造

### パッシブスキルの種類

| 種別 | 解放条件 | 例 |
|------|---------|---|
| レベルパッシブ | 特定レベル到達時 | Lv1, Lv11, Lv41 等 |
| 覚醒パッシブ | 特定覚醒段階到達時 | 覚醒2回目, 3回目 等 |

### 発動タイミングの種類（確認済み）

| タイミング | 説明 |
|-----------|------|
| 試合開始時 | 試合が始まった時点で常時発動 |
| ドリブル技成功時 | ドリブル技が成功するたびに発動 |
| シュートブロック成功時 | ブロックに成功するたびに発動 |
| ペナルティエリア内でシュート時 | 特定場所での行動時に発動 |
| 自身がゴールを入れるたび | 得点のたびに発動（累積バフ） |

### 発動条件のタグ種別（確認済み）

| 条件種別 | 例 |
|---------|---|
| チームタグ指定 | イナズマジャパンタグの味方が3人以上 |
| 属性指定 | 風属性の技のみ有効 |
| 無条件（常時） | 条件なし（試合開始時から発動） |

### スキル接頭辞パターン

| 接頭辞 | 意味 |
|-------|------|
| 【結束】 | 特定チームタグを持つ味方が一定数以上いる場合に発動 |
| 【決定機】 | 特定場面（ペナルティエリア内等）での行動時に発動 |
| 【自得点/累】 | 自身が得点するたびに累積発動 |
| （なし） | 常時発動または単純条件 |

### 効果対象の種類（確認済み）

| 対象 | 例 |
|------|---|
| 自身のみ | 自身のキックを上昇 |
| 特定タグの味方 | 雷門またはイナズマジャパンタグを持つ味方の全パラメータ上昇 |
| 味方全体 | 味方のファウル率を減少 |

## 14. Webアプリ実装時の注意点

1. **属性名の訂正が必要**
   - 旧実装: `'火' | '風' | '木' | '無'` → **誤り**
   - 正しくは: `'火' | '風' | '山' | '林'`

2. **レアリティ表記の訂正が必要**
   - 旧実装: `'S' | 'A' | 'B' | 'C'` → **誤り**
   - 正しくは: `'星1' | '星2' | '星3'`

3. **覚醒段階の表現を変更**
   - 旧実装: 数値（0〜5など）
   - 正しくは: ランク名文字列（`'normal player'` 〜 `'legendary player+'`）

4. **覚醒素材はキャラ固有（覚醒魂）と共通（属性のかけら）の2種類**
   - キャラIDと紐づいた覚醒魂テーブルが必要
   - 属性のかけらは属性別に4種 + 万能1種

5. **パッシブスキルはレベルパッシブと覚醒パッシブの2種類がある**
   - 解放条件（レベル数 or 覚醒段階）をフィールドに追加推奨

6. **1キャラが複数のパッシブスキルを持つ（中間テーブル必要）**

7. **同名キャラが複数レアリティで存在する**（例: 円堂守 星2/星3/日本代表2026）
   - キャラIDは固有。肩書きを含めて識別する設計推奨

8. **著作権配慮**: 公式画像・データの自動取得は行わないこと
